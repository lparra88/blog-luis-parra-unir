Mi Viaje de Aprendizaje

Introducción

¡Hola a todos! Estoy muy emocionado de compartir mi primera entrada de blog con ustedes. En este espacio les voy a contar cómo inicié con la creación de mi propio sitio web y lo publiqué usando GitHub Pages. Espero que esta experiencia pueda ser útil para aquellos que también quieren aprender a crear su propio blog y ponerlo en línea.

Paso 1: Creación del Repositorio en GitHub

El primer paso para crear mi blog fue establecer un repositorio en GitHub. Esto fue clave para almacenar y gestionar todos los archivos del blog de manera organizada. Aquí les dejo cómo lo hice:

  1. Primero, fui a GitHub y me registré (si aún no tienes cuenta, este es el primer paso).
  2. Luego, hice clic en el botón "New" (Nuevo) para crear un repositorio nuevo. Elegí un nombre sencillo, como "blog-luis-parra-unir", y lo hice público para que cualquier persona pudiera acceder al sitio.
  3. Seleccioné la opción "Add a README file" para tener un archivo inicial y agregué una breve descripción sobre el propósito del blog.

Una vez creado el repositorio, estaba listo para empezar a trabajar en los archivos que formarían parte del blog.

Paso 2: Configuración de Git y Visual Studio Code

Para trabajar de manera más cómoda y eficiente, decidí usar Visual Studio Code (VS Code) como mi editor de código y configuré Git para gestionar los cambios en el código de mi proyecto.

  • Instalación de Git: Primero, descargué e instalé Git. Esto me permitió clonar el repositorio y sincronizar los cambios entre mi computadora y GitHub.
  • Configuración de Visual Studio Code: Luego, instalé VS Code. Dentro de VS Code, abrí la terminal e hice la configuración básica de Git:
git config --global user.name "Tu Nombre" git config --global user.email "tu_email@example.com"

Esto es importante porque permite que Git registre quién hace los cambios en cada archivo.

Paso 3: Clonar el Repositorio y Empezar a Trabajar

Una vez que todo estaba listo, cloné el repositorio en mi computadora usando VS Code. Para ello:

  1. Copié la URL del repositorio desde GitHub.
  2. En VS Code, abrí la paleta de comandos (Ctrl + Shift + P o Cmd + Shift + P en Mac) y escribí "Git: Clone".
  3. Pegué la URL y seleccioné una carpeta donde guardar los archivos localmente.

Con el repositorio clonado, comencé a trabajar en los archivos HTML y CSS para crear las páginas de mi blog.

Paso 4: Publicación con GitHub Pages

Finalmente, para publicar mi blog en línea, utilicé GitHub Pages. Esto fue relativamente sencillo:

  1. Fui a la sección Settings del repositorio en GitHub.
  2. Desplacé hacia abajo hasta encontrar la opción "Pages".
  3. Seleccioné la rama "main" y la carpeta "root" para publicar el sitio directamente desde la carpeta principal del repositorio.

Una vez hecho esto, GitHub Pages generó un enlace público para mi sitio, el cual pude compartir con todos ustedes. ¡Fue realmente emocionante ver mi trabajo en línea!

Reflexiones Finales

Este proceso me ayudó a entender mejor cómo se pueden usar herramientas gratuitas y poderosas para crear y publicar contenido en la web. Aunque al principio puede parecer complicado, GitHub Pages es una excelente opción para quienes desean crear un sitio web sin costos y con un control completo sobre el código.

Espero que esta guía haya sido de ayuda para aquellos que también quieren aprender a crear su propio blog y publicarlo. Si tienen alguna pregunta o quieren saber más detalles de algún paso, no duden en dejar un comentario.
